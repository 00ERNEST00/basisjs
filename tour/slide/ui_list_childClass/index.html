Для упрощения код из предыдущего примера, можно использовать свойство childClass при создании узла, которое определяет класс дочерних узлов.

В этом случае в функции appendChild и insertBefore достаточно передать лишь конфиг для нового узла, при этом для создания экземпляра дочернего узла с данным конфгом будет использован класс, указанный в childClass. Если не указывать childClass, то дочерние узлы будут являться экземплярами класса basis.ui.Node.
