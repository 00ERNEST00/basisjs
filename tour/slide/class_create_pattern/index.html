В базом вариант классы basisjs очень похожи на обычные классы.

Но часто бывает, что у класса могут быть много свойств, которые могут быть заданы при создании экземпляра. В этом случае удобно использовать патерн авто расширения экземпляра при создании. Суть его в том, что при создании указывается единственный объект - конфиг. Перед тем как будет вызван init, все свойства из конфига копируются в экземпляр.

За то какой паттерн будет использоваться при создании экземпляров отвечает свойство extendConstructor_ при создании класса. Если свойство не задано, оно наследуется. Если оно true, то используется патерн авто-расширения экземпляра.

Почти у всех классов basisjs extendConstructor_ = true.
